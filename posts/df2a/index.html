随笔

  1. 常见命令：

    docker ps [-a] [--no-trunc] # 查看容器[-a查看所有] [--no-trunc显示完整信息，不截断] [-s,查看占用大小]
docker images # 查看镜像
docker stats [-a] # 查看进程状态

# 后台拉取，正常输出和错误输出都输出到黑洞里
docker pull xxx > /dev/null 2>&1 &

# xxx是[镜像名字:版本号]test-security-api:v2或者镜像ID标识
docker stop xxx(容器名称或id) # 停止容器
docker rm xxx（同上） # 删除容器，-f 强制删除运行中的容器
docker run/restart xxx # 运行/重启 容器
docker start xxx # 启动容器
docker update xxx # 更新容器配置(改不了端口映射，呜呜呜，但是可以改开机启动)
docker exec -it xxx /bin/bash# 进入容器命令行控制台，简写bash也可以
docker rename xx yy # xx容器重命名为yy


docker rmi xxx # 删除镜像

docker -v /1:/c1 -v /2:/c2# 文件映射挂载
docker volume ls #查看卷
docker volume inspect xxx #查看卷详情
docker create volume xxx #创建卷
# docker网络 docker0，进入容器执行ip命令
docker network --help
docker network create xxx #创建网络
docker run --network xxx # 指定网络


docker info
docker logs xxx # 查看容器日志
docker inspect xxx # 查看镜像/容器详细信息

docker build -t ixxx . # 构建镜像，ixxx是镜像名
docker tag [OPTIONS] IMAGE[:TAG] [REGISTRYHOST/][USERNAME/]NAME[:TAG]


# todo docker export（打包我们的容器或者虚拟机）
# https://www.easck.com/cos/2023/0302/1095129.shtml
# docker镜像导入导出（导出我们使用的镜像images）
docker save -o xx.tar 镜像:版本(可以一次打包多个) # output
docker save 镜像:版本 > xx.tar
docker load < xx.tar # input
docker cp 容器标识:目录文件 本地目录文件 # 和cp差不多，就是源文件要加一个容器前缀。

  2. docker exec -it redis redis-cli，进入Redis命令行，而不是Linux命令行

  3. /bin/bash和/bin/sh（bash和sh的区别），Shell编程之/bin/bash和/bin/sh的区别浅析，sh是早期简单的版本，/bin/sh 等价于 /bin/bash --posix。通常使用bash。

    补充：直接使用相对路径 bash 或 sh（会在系统路径echo $PATH中查找）。

  4. docker hub，仓库，可以搜索镜像，包括版本号tag，mysql:8.0.30，还包括一些启动命令和参数配置

  5. 安装docker，官方有教程。

  6. docker阿里云镜像加速，上阿里云控制台，有教程。

  7. 【docker官网有安装docker的教程】，【docker hub仓库有拉去安装运行镜像的教程】，【阿里云有配置镜像加速的教程】

  8. MySQL默认端口是3306，端口映射与MySQL的配置文件（里面配置的服务端口）无关

  9. docker run，配置的好多东西一旦配置完成启动之后就没法简单的改变了（修改配置文件很麻烦的）。

    docker run --name mysql --restart=always -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 -d mysql

    --name 必须紧接在run后面

    -d，后台运行，不占用当前控制台。

    -i允许交互，-t分配一个伪终端（t是terminal终端）。

    -d，守护进程（daemon）。另&也是后台启动。

    –rm 容器退出的时候删除
    -i 交互模式，没有这个单一个t是没法交互的
    -t 当前窗口 tty
    -d 后台运行

  10. Docker 并非是一个通用的容器工具，它依赖于已存在并运行的 Linux 内核环境。Docker 实质上是在已经运行的 Linux 下制造了一个隔离的文件环境。

  11. docker run ，-e配置参数，其实是写入配置文件，user.name转化为USER_NAME（大写，.使用_替代）。

  12. docker数据挂载在外边之后，删除容器后数据也不会丢失。

  13. 复制容器内的文件到Linux里，docker container cp nginx:/etc/nginx .

  14. 多个端口映射 -p linux:docker ，-p 9000:9000 -p 9090:9090 -p 3307:3306。-P docker-port，大写P是随机映射宿主机端口

  15. -v /home/admin/minio/data:/data \ -v /home/admin/minio/config:/root/.minio ，挂载映射，前面是虚拟机路径，后面是镜像路径。

  16. -v 挂载，悬挂，一般容器内的路径（本机路径：目标路径）都在:后面。不需要提前创建文件夹。

    -v不支持直接挂载文件，只能挂载文件夹。想要挂载文件，必须宿主机也要有对应的同名文件。

    docker一个-v只能挂在一个位置，并且创建挂载文件，容器内的会同步成为虚拟机里的文件内容。

  17. 目录挂载：-v /app/nghtml:/usr/share/nginx/html ， 含有/绝对路径或者./相对路径，容器中对应的目录替换使用宿主机中的目录，若宿主机没有则创建空目录。

    卷映射：-v ngconf:/etc/nginx，不带路径前缀，把容器中目录下的所有文件复制出来 。卷在宿主机中的位置：/var/lib/docker/volumes/<volume-name>.

    查看docker卷：docker volume ls。

  18. docker cp my-nginx:/etc/nginx/nginx.conf .，docker复制容器内文件。

  19. docker logs 容器日志

  20. docker安装MySQL示例：docker run --name mysql5 --restart=always -p 3309:3306 -e MYSQL_ROOT_PASSWORD=123456 -d mysql:5.7.23

  21. docker info，可查看镜像加速地址。

  22. dockerfile，制作一个镜像，会运行几条命令，把项目打成jar/war包，然后通过java命令运行这个jar包。当镜像变成容器运行时，就相当于应用被运行了。mvn package时，项目的pom文件要有对应的build设置把依赖的jar包环境也打进去。

  23. java:8,openjdk:8。Docker官方已经弃用java8镜像，想使用jdk1.8的话，可以使用openjdk。解决办法是，将Dockerfile改为 FROM openjdk:8-jre或者openjdk:8。

  24. docker部署（dockerfile部署项目），先打jar包或war包，然后根据dockerfile构建镜像，然后根据镜像运行一个容器。然后这个容器就是部署的服务，访问这个容器。

33. 构建arm的docker镜像，

    安装docker buildx arm环境：

    1. `docker buildx create --platform linux/arm64,linux/arm/v7,linux/arm/v6 --name multiarch-builder`，

    2. `docker buildx inspect multiarch-builder --bootstrap`，
    3. `docker use multiarch-builder`

    使用（基础镜像需要是arm版的）：

    1. `docker buildx build --platform=linux/arm64,linux -t cccsl05/test-curl:v1 . --push`，使用工具重构架构并推送到dockerhub仓库
    2. `docker pull --platform=linux/arm64 cccsl05/test-curl:v1`，从dockerhub仓库指定拉取
    3. `docker run --rm -t cccsl05/test-curl:v6 uname -a`，运行一个 `uname -a`命名看看架构并立即删除创建的容器。-t 是终端，-i是可交互的。这里直接输出了，不需要交互。
    4. `docker tag cccsl05/test-curl:v1 biyi/git-devops:1.0.0-arm-v3`，重新打tag
    5. `docker save biyi/git-devops:1.0.0-arm-v3 > curlv1.tar`，保存为tar文件

### Dockerfile

> 1. `docker build -t test-security-api:v2 .`
>
>    根据 Dockerfile 文件构建docker镜像，
>
>    -t, --tag list  ， Name and optionally a tag in the 'name:tag' format
>
>    最后`.` 指定 dockerfile 所在目录
>
> 2. Dockerfile，就是用来创建一个我们想要的Linux系统
>
> 参考：[Docker Dockerfile | 菜鸟教程 (runoob.com)](https://www.runoob.com/docker/docker-dockerfile.html)。

1. `FROM`，`FROM biyi/jdk11-maven-devops:1.0.0`。

   基础镜像，一般是一个Linux系统加上一些软件，比如Linux上只安装了maven

2. `MAINTAINER <name>`，`MAINTAINER docker_user docker_user@email.com`

   作者信息之类的。（已弃用，推荐使用LABEL指令）

3. `LABEL <key>=<value> <key>=<value> <key>=<value>` ，

   给镜像添加一些元数据（metadata），比如作者信息，`LABEL org.opencontainers.image.authors="runoob"`.

4. `WORKDIR <工作目录路径>`，相当于一直cd到这个目录，没有这个目录会新建。这个目录会成为`docker run`、`docker exec` 进入容器之后的默认目录

   docker build 构建镜像过程中的，每一个 RUN 命令都是新建的一层。只有通过 WORKDIR 创建的目录才会一直存在。

5. `COPY`，`COPY sonar-scanner/sonar-scanner.tgz sonar-scanner.tgz`。

   从当前环境中复制到要构建的镜像中。

   ADD只推荐在解压缩的时候使用。

6. `RUN`，在构建过程中在镜像中执行命令。

   ```dockerfile
   # 每一次RUN都会封装一层，所以尽量使用 && 连接命令
   RUN yum -y install wget \
       && wget -O redis.tar.gz "http://download.redis.io/releases/redis-5.0.3.tar.gz" \
       && tar -xvf redis.tar.gz
   # 压缩包一般使用tar命令解压，使用unzip会遇到各种问题。
   RUN tar -zxvf sonar-scanner.tgz -C /opt

  26. ENV <key1>=<value1> <key2>=<value2>...，ENV PATH=$PATH:/opt/sonar-scanner-4.8.0.2856/bin。

    设置环境变量。Linux中使用env命令可以查看环境变量。

    docker run .. -e，也是指这个ENV，代码中从系统环境变量获取值，环境变量可以在Dockerfile中设置或者运行时传递。

  27. EXPOSE <port> [<port>...]，镜像是一个Linux系统，会有防火墙拦截端口访问不到，设置开放端口可以给外部访问。暴露多个EXPOSE 9000 9001。

    • 在运行时使用随机端口映射时，也就是 docker run -P 时，会自动随机映射 EXPOSE 的端口。-p 是指定映射。不写应该就是不映射

  28. VOLUME ["<路径1>", "<路径2>"...]，【数据/文件持久化】必须挂载到宿主机的卷，不会因为容器消失里面的东西就消失。

    VOLUME <路径>，定义匿名数据卷。在启动容器时忘记挂载数据卷，会自动挂载到匿名卷。

    在启动容器 docker run 的时候，我们可以通过 -v 参数修改挂载点。

    宿主机和docker没有路径目录都会创建一个，宿主机使用相对路径需要显式加上./，-v 宿主机目录 : docker目录

    -v挂载目录，不要求容器必须存在该目录，会自动创建，比如 -v /app/security-api/config:/config。

  29. CMD 指定容器创建时的默认命令。（可以被覆盖）

    ENTRYPOINT 设置容器创建时的主要命令。（不可被覆盖）

    若一个dockerfile中有多个CMD命令行，则只有最后一个会生效。（不信问AI：一个dockerfile中的多个CMD命令，是不是只有最后一个CMD会生效），

    想要在一个容器启动时运行多个命令，你可以将它们放在同一个CMD指令中，使用&&来连接它们

  30. ARG <参数名>[=<默认值>]，

    构建参数，与 ENV 作用一致。不过作用域不一样。ARG 设置的环境变量仅对 Dockerfile 内有效，也就是说只有 docker build 的过程中有效，构建好的镜像内不存在此环境变量。

    构建命令 docker build 中可以用 --build-arg <参数名>=<值> 来覆盖。

  31. CMD ["/bin/sh"]，

    docker run 运行容器时的默认运行命令，多个CMD会被覆盖。

    docker run --rm -t git-devops:1.0.0 uname -a，这里的uname -a就是run运行的命令。

  32. ENTRYPOINT ["sh", "/entrypoint.sh"]，

    类似于 CMD 指令，但其不会被 docker run 的命令行参数指定的指令所覆盖，而且run这些命令行参数（上面的uname -a）会被当作参数送给 ENTRYPOINT 指令指定的程序。

    FROM nginx
...
ENTRYPOINT ["nginx", "-c"] # 定参
CMD ["/etc/nginx/nginx.conf"] # 变参

    1、不传参运行，使用dockerfile中的 CMD
    $ docker run nginx:test 

    nginx -c /etc/nginx/nginx.conf

    2、传参运行，使用命令行传递的 /etc/nginx/new.conf
    $ docker run nginx:test /etc/nginx/new.conf

    nginx -c /etc/nginx/new.conf
    # 放一个curl的entrypoint.sh

if [ "${1#-}" != "${1}" ] || [ -z "$(command -v "${1}")" ]; then
  set -- curl "$@"
