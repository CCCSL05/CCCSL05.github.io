变量

  1. 声明变量，variable_name=value，等号两侧避免使用空格。

    还可以用语句给变量赋值，如：

    for file in `ls /etc`
或
for file in $(ls /etc)

  2. 使用变量，echo $your_name 或 echo ${your_name}。花括号明确指定边界。

    直接写 $defCmd，会把变量的值当成命令执行。

  3. 只读变量，readonly myUrl="https://www.google.com"。

  4. 删除变量，unset variable_name。

  5. 读取变量：read -p "请输入密码:" -n 6 -t 5 -s password。

  6. 整数变量，declare -i my_integer=42。一些Shell中使用 declare 或 typeset 命令来声明整数变量。

  7. 数组变量，my_array=(1 2 3 4 5)。括号里的各个元素的空格可以使用换行符代替。

    1. 单独定义数组的各个分量：array_name[0]=value0，array_name[n]=valuen，下标可以不连续。

    2. 读取数组某个元素：${数组名[下标]}。

    3. 读取数组全部元素：echo ${array_name[@]}。

    4. 获取数组的长度：length=${#array_name[@]}，length=${#array_name[*]}，

    5. 关联数组：Map 键值对。-A 选项就是用于声明一个关联数组。

      declare -A site=(["google"]="www.google.com" ["runoob"]="www.runoob.com" ["taobao"]="www.taobao.com")

declare -A site
site["google"]="www.google.com"
site["runoob"]="www.runoob.com"
site["taobao"]="www.taobao.com"
echo ${site["runoob"]}

    6. 6

  8. 环境变量，echo $PATH。

  9. 特殊变量，例如 $0 表示脚本的名称，$1, $2, 等表示脚本的参数。$#表示传递给脚本的参数数量，$? 表示上一个命令的退出状态等。

  10. 字符串变量，使用 单引号 或 双引号。

    1. 单引号里面统统原样输出。

      双引号里面可以使用变量、转义符、特殊语法。

    2. 拼接字符串：使用 单引号 或 双引号 拼接，中级不能有空格分隔。类似php中的.，Java中默认的+连接。

      echo greeting="hello1, "$your_name，相当于Java中 greeting="hello1, " + $your_name。

      echo greeting='hello2, '$your_name'.ok.'，相当于Java中 greeting="hello2, " + $your_name + ".ok."。

    3. 获取字符串长度：echo ${#name}。

    4. 子字符串：echo ${string:1:4}，从字符串第 2 个字符开始截取 4 个字符。下标0开始，截取n个字符。

    5. 查找子字符串： echo `expr index “$name” zhang` ，下标从1开始。

注释

  1. 单行注释，#

  2. 多行注释

:<<EOF
注释内容...
注释内容...
注释内容...
EOF

:<<'COMMENT'
这是注释的部分。
可以有多行内容。
COMMENT

:<<''
注释内容...
注释内容...
注释内容...
'

:<<!
注释内容...
注释内容...
注释内容...

: 是一个空命令，用于执行后面的 Here 文档，<<'EOF' 表示开启 Here 文档，COMMENT 是 Here 文档的标识符，在这两个标识符之间的内容都会被视为注释，不会被执行。
EOF 也可以使用其他符号。

: '
这是注释的部分。
可以有多行内容。
'

使用冒号 : 命令，并用单引号 ' 将多行内容括起来。由于冒号是一个空命令，这些内容不会被执行。

格式为：: + 空格 + 单引号。

Shell 传递参数

  1. 其中 $1 表示第一个参数，$2 表示第二个参数，依此类推。
  2. $* 与 $@ 区别：
    • 相同点：都是引用所有参数。
    • 不同点：只有在双引号中体现出来。假设在脚本运行时写了三个参数 1、2、3，则 " $* " 等价于 “1 2 3”（传递了一个参数），而 "$@" 等价于 “1” “2” “3”（传递了三个参数）。
参数处理 说明
$# 传递到脚本的参数个数
$* 以一个单字符串显示所有向脚本传递的参数。 如”$*“用「”」括起来的情况、以”$1 $2 … $n“的形式输出所有参数。
$$ 脚本运行的当前进程ID号
$! 后台运行的最后一个进程的ID号
$@ 与$*相同，但是使用时加引号，并在引号中返回每个参数。 如”$@“用「”」括起来的情况、以”$1“ “$2“ … “$n“ 的形式输出所有参数。
$- 显示Shell使用的当前选项，与 set命令 功能相同。
$? 显示最后命令的退出状态。0表示没有错误，其他任何值表明有错误。

自增和自减操作符

#!/bin/bash

# 初始化变量
num=5
echo "初始值: $num"

# 自增
let num++
echo "自增后: $num"
# 自减
let num--
echo "自减后: $num"

# 使用 $(( ))
num=$((num + 1))
echo "使用 $(( )) 自增后: $num"
num=$((num - 1))
echo "使用 $(( )) 自减后: $num"

# 使用 expr
num=$(expr $num + 1)
echo "使用 expr 自增后: $num"
num=$(expr $num - 1)
echo "使用 expr 自减后: $num"

# 使用 (( ))
((num++))
echo "使用 (( )) 自增后: $num"
((num--))
echo "使用 (( )) 自减后: $num"

echo

echo string。 echo 会自动添加换行符。

  1. 显示普通字符串：echo "It is a test"。

    可以省略双引号：echo It is a test。

  2. 显示转义字符：echo "\"It is a test\""。输出 "It is a test"。同样，双引号也可以省略

  3. 显示变量：read name; echo "$name It is a test".

  4. 显示换行：echo -e "OK! \nIt is a test" 。-e 开启转义。

    \n换行，\c 不换行：echo -e "OK! \c"; echo "It is a test"

  5. 显示结果定向至文件：echo "It is a test" > myfile。

  6. 原样输出：使用单引号

  7. 显示命令执行结果：echo `date`

printf

printf 命令模仿 C 程序库（library）里的 printf() 程序。

默认的 printf 不会自动添加换行符，我们可以手动添加 \n。

格式：printf format-string [arguments...]

  • %s：字符串
  • %d：十进制整数
  • %f：浮点数
  • %c：字符
  • %x：十六进制数
  • %o：八进制数
  • %b：二进制数
  • %e：科学计数法表示的浮点数

%-10s 指一个宽度为 10 个字符（**-** 表示左对齐，没有则表示右对齐），任何字符都会被显示在 10 个字符宽的字符内，如果不足则自动以空格填充，超过也会将内容全部显示出来。

%-4.2f 指格式化为小数，其中 .2 指保留 2 位小数。

示例：printf "%s %s %s\n" a b c d e f g h i j

test 命令

linux shell中test命令和[] 的区别

在 Linux shell 中，test 命令和 []（也称为方括号测试）实际上是等价的。[] 实际上是 test 命令的快捷方式，并且它们都被用于评估条件表达式。

但是，从语法和使用上来说，它们之间有一些细微的差异：

  1. test 命令：

test 命令使用 -a 操作符进行与操作，使用 -o 操作符进行或操作，不支持 ||、&& 这种写法的操作符。

另外，在 bash 中，小括号具有特殊含义，(cmd) 表示启动一个子 shell 来执行 cmd 命令。

所以这里的 ( 和 ) 要用 \ 转义字符、或者引号来去掉它们的特殊含义，避免被当成命令替换（Command substitution）来处理。

if test -e file.txt; then
    echo "file.txt exists"
fi
  1. []（方括号测试）：

该命令是 test 命令的同义词，它对条件表达式的判断结果和 test 命令完全一样。

if [ -e file.txt ]; then
    echo "file.txt exists"
fi

注意，[ 和 ] 实际上是 shell 中的内建命令 test 的快捷方式，并且它们需要与其他内容用空格分隔。

  1. [[ ]]（双方括号测试）：

除了 test 和 []，Bash 还支持 [[ ]]，这是 Bash 特有的，并提供了更强大的功能和更少的错误（例如，无需对变量进行引号处理以防止文件名扩展和单词拆分）。

if [[ -e "$file_name" ]]; then
    echo "$file_name exists"
fi

在 Bash 中，当您可以使用 [[ ]] 时，通常建议使用它，因为它提供了更多的功能和更好的错误处理。但是，请注意，[[ ]] 不是 POSIX 标准的一部分，因此它可能不会在所有的 shell 中都可用。

总之，test 和 [] 在功能上是等价的，但 [[ ]] 在 Bash 中提供了更多的功能和更好的错误处理。
