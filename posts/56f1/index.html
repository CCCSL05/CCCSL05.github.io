docker安装ollama

  1. 拉取docker镜像，docker.io/ollama/ollama:0.2.5 - 镜像下载 | docker.io (aityp.com).

    docker pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/docker.io/ollama/ollama:0.2.5
docker tag  swr.cn-north-4.myhuaweicloud.com/ddn-k8s/docker.io/ollama/ollama:0.2.5  docker.io/ollama/ollama:0.2.5

  2. 启动镜像，端口映射11434。

    进入容器，拉取运行模型，library (ollama.com)。

    我选择的是qwen2，在容器内部运行 ollama run qwen2。qwen2 (ollama.com)。

  3. 运行后即可发送消息交互。

  4. API 参考 - Ollama中文网。

  5. ==win拉取模型失败是因为权限不够==。管理员模式运行powershell可以ollama pull xxx 下载模型

open-webui安装

  1. GitHub - open-webui/open-webui: User-friendly WebUI for LLMs (Formerly Ollama WebUI)。

  1. 拉取镜像，

    docker pull swr.cn-north-4.myhuaweicloud.com/ddn-k8s/ghcr.io/open-webui/open-webui:main
docker tag  swr.cn-north-4.myhuaweicloud.com/ddn-k8s/ghcr.io/open-webui/open-webui:main  ghcr.io/open-webui/open-webui:main

  2. 启动镜像：

    # 本地
docker run -d -p 3000:8080 --add-host=host.docker.internal:host-gateway -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:main
# 或者 替换OLLAMA_BASE_URL参数值
docker run -d -p 3000:8080 -e OLLAMA_BASE_URL=https://example.com -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:main

LLaMA-Factory 微调

【大模型开发 】 一文搞懂Fine-tuning（大模型微调）_finetuning大模型-CSDN博客。

ollama 使用自己的微调模型_ollama 微调-CSDN博客.

LLM 大模型学习必知必会系列(十二)：VLLM性能飞跃部署实践：从推理加速到高效部署的全方位优化-腾讯云开发者社区-腾讯云 (tencent.com)。

最好是Linux环境，拥有GPU

代码地址：https://ghproxy.net/https://github.com/hiyouga/LLaMA-Factory.git

文档：README_zh.md

  1. pytorch环境 2.1.2，Previous PyTorch Versions | PyTorch。python=3.8

    # conda
conda install pytorch==2.1.2 torchvision==0.16.2 torchaudio==2.1.2
# pip
pip install torch==2.1.2 torchvision==0.16.2 torchaudio==2.1.2
pip install torch==2.0.1 torchvision==0.15.2 torchaudio==2.0.2

  2. pip install -r requirements.txt

  3. python setup.py install，build/bin目录 llamafactory-cli命令

  4. python src/webui.py，启动webui页面，填写关键信息，最后预览命令。

  5. 微调，llamafactory-cli train --stage sft --do_train True --model_name_or_path /mnt/workspace/.cache/modelscope/hub/qwen/Qwen1___5-0___5B-Chat --preprocessing_num_workers 16 --finetuning_type lora --template default --flash_attn auto --dataset_dir data --dataset ruozhiba_qa --cutoff_len 1024 --learning_rate 5e-04 --num_train_epochs 6.0 --max_samples 100000 --per_device_train_batch_size 2 --gradient_accumulation_steps 8 --lr_scheduler_type cosine --max_grad_norm 1.0 --logging_steps 5 --save_steps 100 --warmup_steps 0 --optim adamw_torch --packing False --report_to none --output_dir saveswen1.5-0.5Borarain_2024-08-05-14-18-19 --bf16 True --plot_loss True --ddp_timeout 180000000 --include_num_input_tokens_seen True --lora_rank 8 --lora_alpha 16 --lora_dropout 0 --lora_target all。

  6. 合并（导出），llamafactory-cli export --model_name_or_path /mnt/workspace/.cache/modelscope/hub/qwen/Qwen1___5-0___5B-Chat --adapter_name_or_path /mnt/workspace/LLaMA-Factory/savesQwen1.5-0.5Bloratrain_2024-08-05-14-18-19 --template default --finetuning_type lora --export_dir megred-model-path2 --export_size 2 --export_device cpu --export_legacy_format False。

    llamafactory-cli export --help，查看帮助。

from modelscope import AutoModelForCausalLM, AutoTokenizer
device = "cpu" # the device to load the model onto
 
model = AutoModelForCausalLM.from_pretrained(
    "qwen/Qwen1.5-0.5B-Chat",
    torch_dtype="auto",
    device_map="auto"
)
tokenizer = AutoTokenizer.from_pretrained("qwen/Qwen1.5-0.5B-Chat")
 
prompt = "Give me a short introduction to large language model."
prompt = "你是谁"
prompt = "青海5月30的天气"
prompt = "你好呀"

messages = [
    # {"role": "system", "content": "You are a helpful assistant."},
    {"role": "system", "content": "请用二次元可爱语气和我说话."},
    {"role": "user", "content": prompt}
]
text = tokenizer.apply_chat_template(
    messages,
    tokenize=False,
    add_generation_prompt=True
)
model_inputs = tokenizer([text], return_tensors="pt").to(device)
 
generated_ids = model.generate(
    model_inputs.input_ids,
    max_new_tokens=512
)
generated_ids = [
    output_ids[len(input_ids):] for input_ids, output_ids in zip(model_inputs.input_ids, generated_ids)
]
 
response = tokenizer.batch_decode(generated_ids, skip_special_tokens=True)[0]

print(response)

大模型RAG是怎么实现的，怎么和大模型交互的

大模型RAG（Retrieval-Augmented Generation，检索增强生成）的实现及其与大模型的交互过程可以概括为以下几个关键步骤：

RAG的实现过程

索引（Indexing）：

将文档库分割成较短的Chunk（片段），这些Chunk可以是句子、段落或根据特定策略生成的更复杂的文本块。
使用编码器（如BERT、Transformer等）将Chunk编码成向量，并存储在向量数据库中。

检索（Retrieval）：

当用户提出问题时，将问题也编码成向量。
使用向量检索技术（如Faiss、nmslib等）在向量数据库中检索与问题向量最相似的Chunk。
根据相似度排序，选取最相关的前k个Chunk作为候选答案的上下文。

生成（Generation）：

将原始问题和检索到的Chunk一起输入到大型语言模型（LLM）中。
LLM根据输入的上下文和问题生成最终答案。
RAG与大模型的交互

上下文注入：

RAG通过检索到的相关Chunk为LLM提供了额外的上下文信息。
这些上下文信息有助于LLM生成更准确、更相关的回答，减少“幻觉”现象的发生。

动态更新：

由于RAG可以实时从外部知识库中检索信息，因此它可以帮助LLM对自身知识进行动态更新。
这使得LLM在回答需要当前信息的查询时更加实用和可信。

可追溯性：

RAG增强了内容生成的可追溯性，因为可以明确地知道LLM模型根据哪些信息得出的回答。
这有助于解决可解释性、信息溯源和信息验证等问题。
示例流程

以一个简单的问答系统为例，流程可能如下：

用户输入问题：“什么是RAG？”
系统将问题编码成向量，并在向量数据库中检索与“RAG”相关的Chunk。
检索到相关Chunk后，将其与原始问题一起输入到LLM中。
LLM根据输入的上下文和问题生成答案：“RAG是检索增强生成技术，它通过检索外部知识库中的相关信息来增强大型语言模型的生成能力。”

总的来说，RAG通过索引、检索和生成三个步骤实现了与大模型的交互，从而提高了生成内容的准确性和相关性。
