前言

Optional 类是一个可以为null的容器对象。如果值存在则isPresent()方法会返回true，调用get()方法会返回该对象。

Optional 是个容器：它可以保存类型T的值，或者仅仅保存null。Optional提供很多有用的方法，这样我们就不用显式进行空值检测。

Optional 类的引入很好的解决空指针异常。

主要是拿到对象深层的值，并且避免空指针异常。要么我们要找的值层层存在，要么走orElse的逻辑使用默认值。

小结

Optional提供的常用操作有：

过程操作：map()，flatMap()，filter()；

末尾操作：ifPresent()，isPresent()，get()，orElse()，orElseGet()，orElseThrow()；

创建Optional


Optional<Object> empty = Optional.empty();

// userVO 为 null 会抛 NPE
Optional.of(userVO);

// userVO 为 null 不会抛 NPE
Optional.ofNullable(userVO);

区别：


// 存储的值为空，调用get()方法会发生NPE
Optional.empty();

// 只对第一个参数 userVO 判断是否为null，若为null，则抛出 NPE (NullPointerException) 
String s1 = Optional.of(userVO).map(UserVO::getUser1).map(User1::getName).orElse("100"); // 100

// 允许 uerVO 为null
String s2 = Optional.ofNullable(userVO).map(UserVO::getUser1).map(User1::getName).orElse("100");

过程操作

过程操作（或者说叫转换操作），会对当前Optional流进行处理（更改算法规则，所以很快），返回一个新的Optional流。

流是一次性的，操作后会改变，当金星收尾操作后，该流将不复存在。

Map

接收 Optional 包裹的值，若为空执不执行内部方法。

返回一个值，被 Optional.ofNull 包裹，可以继续操作。

// 字符串长度若大于10，则截取前十个字符
String str="abc1234567890";
Optional<String> a = Optional.ofNullable(str).map(s1 -> {
    if (s1.length() > 10) {
        return Optional.ofNullable(s1.substring(0, 10));
    }
    // return null; // 返回null会让Optional包裹的对象为空，也就是empty，isPresent判断为false
    return s1;
});
/**
 * If a value is present, apply the provided mapping function to it, and if the result is non-null, 
 * return an Optional describing the result. Otherwise return an empty Optional.
 */
public<U> Optional<U> map(Function<? super T, ? extends U> mapper) {
    Objects.requireNonNull(mapper);
    if (!isPresent())
        return empty();
    else {
        return Optional.ofNullable(mapper.apply(value));
    }
}

flatMap

和 map 作用相同，只不过返回值不一样。

flatMap 返回的是 Optional 【Optional.of(xxx) 或者 Optional.ofNull(xxx)】

String str="abc1234567890";
Optional<String> a = Optional.ofNullable(str).flatMap(s1 -> {
    if (s1.length() > 10) {
        return Optional.ofNullable(s1.substring(0, 10));
    }
    // return Optional.empty();
    return Optional.ofNullable(null);
});
public<U> Optional<U> flatMap(Function<? super T, Optional<U>> mapper) {
    Objects.requireNonNull(mapper);
    if (!isPresent())
        return empty();
    else {
        return Objects.requireNonNull(mapper.apply(value));
    }
}

filter

如果值存在，并且这个值匹配给定的 predicate，返回一个Optional用以描述这个值，否则返回一个空的Optional。

如果值存在，则进行条件判断。判断成功则返回该值继续使用该值进行下一步操作，判断失败则返回空。

List<String> stringList = new ArrayList<>();
stringList.add("777");
Optional<List<String>> strings1 = Optional.ofNullable(stringList).filter(list -> list.size() > 0);

public Optional<T> filter(Predicate<? super T> predicate) {
    Objects.requireNonNull(predicate);
    if (!isPresent())
        return this;
    else
        return predicate.test(value) ? this : empty();
}

末尾操作

没有后续操作

ifPresent

如果值存在则使用该值调用 consumer , 否则不做任何事情。

public Optional<T> filter(Predicate<? super T> predicate) {
    Objects.requireNonNull(predicate);
    if (!isPresent())
        return this;
    else
        return predicate.test(value) ? this : empty();
}

isPresent

返回 boolean，如果值存在则方法会返回true，否则返回 false。

/**
 * Return {@code true} if there is a value present, otherwise {@code false}.
 *
 * @return {@code true} if there is a value present, otherwise {@code false}
 */
public boolean isPresent() {
    return value != null;
}

get

如果在这个Optional中包含这个值，返回值，否则抛出异常：NoSuchElementException。

返回一个值。常和 isPresent 结合使用，先判断是否有值。

public T get() {
    if (value == null) {
        throw new NoSuchElementException("No value present");
    }
    return value;
}

orElse

如果存在该值，返回值， 否则返回 other。

返回值，若值为空则返回默认值。

// 接受一个对象，所以这里传递方法调用还是构造方法，都会执行得到对象实例并传递进去
// 一般传递一个已知的常量值建议使用这个
    public T orElse(T other) {
        return value != null ? value : other;
    }

orElseGet

和 orElse 类似，只不过接收的不是值，而是匿名函数，通过函数的返回值当成默认值。

// 传递一个函数进来，只有当条件为false时才会调用方法。
// 一个需要计算的值建议使用这个
    public T orElseGet(Supplier<? extends T> other) {
        return value != null ? value : other.get();
    }
// 当数据量大的时候使用 orElseGet 可以避免创建对象。

TestUser value = new TestUser();
//        TestUser value = null;
// 构造函数打印得知，当value !=null 的时候也会创建，而调用方法不会
TestUser testUser = value != null ? value : new TestUser(); 
System.out.println("----------------");
TestUser testUser2 = value != null ? value : getTestUser();

orElseThrow

前提无异常走到最后一步。如果存在该值，返回包含的值，否则抛出由 Supplier 继承的异常。

public <X extends Throwable> T orElseThrow(Supplier<? extends X> exceptionSupplier) throws X {
    if (value != null) {
        return value;
    } else {
        throw exceptionSupplier.get();
    }
}

public final class Optional<T> {
    //Null指针的封装
    private static final java.util.Optional<?> EMPTY = new java.util.Optional<>();
    //内部包含的值对象
    private final T value;
    private Optional() ;
    //返回EMPTY对象
    public static<T> java.util.Optional<T> empty() ;
    //构造函数，但是value为null，会报NPE
    private Optional(T value);
    //静态工厂方法，但是value为null，会报NPE
    public static <T> java.util.Optional<T> of(T value);
    //静态工厂方法，value可以为null
    public static <T> java.util.Optional<T> ofNullable(T value) ;
    //获取value，但是value为null，会报NoSuchElementException
    public T get() ;
    //返回value是否为null
    public boolean isPresent();
    //如果value不为null，则执行consumer式的函数，为null不做事
    public void ifPresent(Consumer<? super T> consumer) ;
     //过滤，如果value不为null，则根据条件过滤，为null不做事
    public java.util.Optional<T> filter(Predicate<? super T> predicate) ;
     //转换，在其外面封装Optional，如果value不为null，则map转换，为null不做事。映射返回值对象
    public<U> java.util.Optional<U> map(Function<? super T, ? extends U> mapper);
     //转换，如果value不为null，则map转换，为null不做事。映射返回值对象的Optional
    public<U> java.util.Optional<U> flatMap(Function<? super T, java.util.Optional<U>> mapper) ;
    //value为null时，默认提供other值
    public T orElse(T other);

    public T orElseGet(Supplier<? extends T> other);
      //value为null时，默认提供other值
    public <X extends Throwable> T orElseThrow(Supplier<? extends X> exceptionSupplier) ;
}
public<U> Optional<U> map(Function<? super T, ? extends U> mapper) {
    Objects.requireNonNull(mapper);
    if (!isPresent())
